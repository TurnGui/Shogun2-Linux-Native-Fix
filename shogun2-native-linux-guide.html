Total War: SHOGUN 2
Native Linux Fix

How to run the abandoned Feral Interactive native Linux port on modern Ubuntu-based systems (glibc 2.34+, Ubuntu 22.04+).

Before you start: If you just want to play Shogun 2 on Linux, use Proton GE instead. It works better, takes 5 minutes to set up, and multiplayer works. This guide exists for people who specifically want the native Linux binary running — out of curiosity, principle, or stubbornness.

What works after this fix

Single-player (campaigns, battles, custom battles)✓ works fully
Steam achievements✓ works
Steam cloud saves✓ works
Multiplayer battles✗ broken (TLS 1.3)
Online leaderboards✗ broken (TLS 1.3)

Multiplayer is broken because we compile OpenSSL 1.0.2 (needed for the old sonames the binary expects), and modern Sega servers reject connections from OpenSSL 1.0.x. There is no easy fix for this.

Requirements

  • required Ubuntu 22.04+ / Linux Mint 21+ / Pop!_OS 22.04+ (or any Ubuntu-based distro)
  • required Steam native, not Flatpak — check: which steam → /usr/games/steam
  • required Shogun 2 installed on the linux-pre-2022-update beta branch
  • required Internet connection (to download OpenSSL and libcurl sources)
  • NVIDIA only 32-bit GL driver — the automated script handles this

Option A — Automated script (recommended)

Download and run the script. It handles everything: compiling OpenSSL, building libcurl, creating the libgconf stub, installing apt dependencies, and printing the exact Steam launch option to use.

download
wget https://github.com/yourname/shogun2-native-fix/raw/main/shogun2-native-fix.sh
chmod +x shogun2-native-fix.sh
preview without doing anything
./shogun2-native-fix.sh --dry-run
run everything
./shogun2-native-fix.sh --all

At the end, the script prints the exact Steam launch option to paste. Skip to the Steam setup section below.

If something fails: run ./shogun2-native-fix.sh --diagnose to see what the game is complaining about. Each phase can be re-run independently with --phase N.

Option B — Manual step by step

If you prefer to understand what's happening, or the script doesn't work on your system, follow these steps.

PHASE 1 Install Steam native + build dependencies ~2 min

If you have Steam Flatpak, remove it and install the native package:

flatpak uninstall com.valvesoftware.Steam  # if you have Flatpak Steam
sudo apt install steam-installer

Install build tools and 32-bit system libraries:

sudo apt install gcc gcc-multilib make wget perl pkg-config build-essential \
  libgl1-mesa-dri:i386 libgl1:i386 libc6:i386 \
  libopenal1:i386 libnss3:i386 libnspr4:i386 \
  libsdl2-ttf-2.0-0:i386 libgdk-pixbuf-2.0-0:i386 \
  libxss1:i386 libxtst6:i386 libgtk2.0-0t64:i386

If you have an NVIDIA GPU, also install the 32-bit GL library. Replace 595 with your actual driver version (nvidia-smi --query-gpu=driver_version --format=csv,noheader):

sudo apt install libnvidia-gl-595:i386
PHASE 2 Build libc_mprotect.so ~10 sec

Modern kernels enforce W^X (write XOR execute). The game requests both writable and executable memory, which crashes. This shim intercepts mprotect() and silently adds PROT_READ to PROT_EXEC-only requests.

mkdir -p ~/.local/share/shogun2-native-fix/lib32
mkdir -p ~/.local/share/shogun2-native-fix/src

cat > ~/.local/share/shogun2-native-fix/src/libc_mprotect.c <<'EOF'
#define _GNU_SOURCE
#include <sys/mman.h>
#include <unistd.h>
#include <sys/syscall.h>
int mprotect(void *addr, size_t len, int prot) {
    if (prot == PROT_EXEC) prot |= PROT_READ;
    return syscall(__NR_mprotect, addr, len, prot);
}
EOF

gcc -m32 -shared -fPIC -O2 \
    -o ~/.local/share/shogun2-native-fix/lib32/libc_mprotect.so \
    ~/.local/share/shogun2-native-fix/src/libc_mprotect.c

file ~/.local/share/shogun2-native-fix/lib32/libc_mprotect.so
# Should say: ELF 32-bit LSB shared object, Intel 80386
PHASE 3 Build OpenSSL 1.0.2u (32-bit) ~5-8 min

The Feral binary was compiled against very old OpenSSL sonames (libssl.so.37, libcrypto.so.36). No modern distro ships these. We compile OpenSSL 1.0.2u ourselves.

Critical lessons learned from field testing:
1. Must use no-asm — GCC modern has a duplicate symbol conflict (bn_sub_part_words) when assembler is enabled
2. Use CFLAG= not CFLAGS= on the make command line — OpenSSL 1.0.2 uses CFLAG internally
3. Must pass -Wl,-z,noexecstack — modern ld treats missing .note.GNU-stack as an error
4. All three steps (Configure, make depend, make) must run in the same directory 
cd ~/.local/share/shogun2-native-fix/src
INSTALL=~/.local/share/shogun2-native-fix
LIB32=$INSTALL/lib32

# Download (SHA256: ecd0c6ffb493dd06707d38b14bb4d8c2288bb7033735606569d8f90f89669d16)
wget -O openssl-1.0.2u.tar.gz \
  https://ftp.nluug.nl/security/openssl/openssl-1.0.2u.tar.gz

tar -xzf openssl-1.0.2u.tar.gz
cd openssl-1.0.2u

# Configure — note: no-asm is required
./Configure linux-elf shared no-asm \
    --prefix="$INSTALL/openssl-install" \
    --openssldir="$INSTALL/openssl-install/ssl" \
    -Wl,-rpath="$LIB32" \
    -Wl,-z,noexecstack

make depend

# Build — note CFLAG= not CFLAGS=
make -j$(nproc) \
    CFLAG="-Wno-error -fPIC -m32 -DOPENSSL_PIC -DOPENSSL_THREADS \
           -D_REENTRANT -DDSO_DLFCN -DHAVE_DLFCN_H -DL_ENDIAN -O3 -Wall"

# Copy and create old soname symlinks
cp -P libssl.so* libcrypto.so* "$LIB32/"
cd "$LIB32"
ln -sf libssl.so.1.0.0 libssl.so.37
ln -sf libcrypto.so.1.0.0 libcrypto.so.36
PHASE 4 Build libcurl 7.40 (linked against our OpenSSL) ~3 min

The Feral binary needs CURL_OPENSSL_4 symbol version, which the system libcurl no longer provides. We compile our own, linked against the OpenSSL we just built.

cd ~/.local/share/shogun2-native-fix/src
OPENSSL_SRC=~/.local/share/shogun2-native-fix/src/openssl-1.0.2u
LIB32=~/.local/share/shogun2-native-fix/lib32

wget -O curl-7.40.0.tar.gz https://curl.se/download/curl-7.40.0.tar.gz
tar -xzf curl-7.40.0.tar.gz
cd curl-7.40.0

CFLAGS="-m32 -I$OPENSSL_SRC/include" \
LDFLAGS="-m32 -L$OPENSSL_SRC -Wl,-rpath=$LIB32" \
LIBS="-ldl" \
./configure \
    --prefix=~/.local/share/shogun2-native-fix/curl-install \
    --with-ssl="$OPENSSL_SRC" \
    --disable-static --enable-shared \
    --host=i686-pc-linux-gnu

make -j$(nproc)
cp -P lib/.libs/libcurl.so* "$LIB32/"
PHASE 5 Build libgconf-2.so.4 stub ~5 sec

GConf was removed from Ubuntu 22.04+. The Feral CEF browser (libcef.so) references it. These stubs satisfy the dynamic linker — the functions are never actually called during single-player.

cat > ~/.local/share/shogun2-native-fix/src/gconf_stub.c <<'EOF'
#include <stdlib.h>
void gconf_client_get_default(void) {}
void gconf_client_get_bool(void) {}
void gconf_client_get_string(void) {}
void gconf_client_get_int(void) {}
void gconf_client_get_list(void) {}
void gconf_client_get(void) {}
void gconf_client_set_bool(void) {}
void gconf_client_set_string(void) {}
void gconf_client_set_int(void) {}
void gconf_client_add_dir(void) {}
void gconf_client_remove_dir(void) {}
void gconf_client_notify_add(void) {}
void gconf_client_notify_remove(void) {}
void gconf_entry_get_key(void) {}
void gconf_entry_get_value(void) {}
void gconf_entry_free(void) {}
void gconf_value_free(void *val) { free(val); }
void gconf_value_get_bool(void) {}
void gconf_value_get_string(void) {}
void gconf_value_get_int(void) {}
void gconf_init(void) {}
void gconf_error_quark(void) {}
EOF

gcc -m32 -shared -fPIC -O2 \
    -o ~/.local/share/shogun2-native-fix/lib32/libgconf-2.so.4 \
    ~/.local/share/shogun2-native-fix/src/gconf_stub.c
PHASE 6 Copy private_symbol_hack.so from game ~5 sec

Feral already included a fix for the __libc_dlopen_mode glibc 2.34+ issue in the game files. We just need to copy it to a path without spaces, because the directory "Total War SHOGUN 2" has spaces that break LD_PRELOAD parsing.

GAME=~/.steam/debian-installation/steamapps/common/"Total War SHOGUN 2"

cp "$GAME/lib/private_symbol_hack.so" \
   ~/.local/share/shogun2-native-fix/lib32/

# Verify
file ~/.local/share/shogun2-native-fix/lib32/private_symbol_hack.so
# Should say: ELF 32-bit LSB shared object, Intel 80386

Steam setup

01
Switch to the native Linux branch

Right-click Shogun 2 in Steam → Properties → Betas tab.
Select linux-pre-2022-update from the dropdown.
Wait for Steam to download this version (~same size as current).

02
Disable Proton

Properties → Compatibility tab.
Uncheck "Force the use of a specific Steam Play compatibility tool".
We want the native Linux binary, not Proton.

03
Set the launch option

Properties → General → Launch Options. Paste this exactly (one long line — replace /home/turn with your actual home directory):

SteamAppId=34330 GameAppId=34330 GAME_LAUNCH_PREFIX="env LD_LIBRARY_PATH=/home/turn/.local/share/shogun2-native-fix/lib32:../lib/i686" LD_PRELOAD="/home/turn/.local/share/shogun2-native-fix/lib32/private_symbol_hack.so:/home/turn/.local/share/shogun2-native-fix/lib32/libc_mprotect.so:/home/turn/.local/share/shogun2-native-fix/lib32/libcurl.so.4.3.0:/home/turn/.local/share/shogun2-native-fix/lib32/libssl.so.1.0.0:/home/turn/.local/share/shogun2-native-fix/lib32/libcrypto.so.1.0.0" %command%

The automated script prints this with your actual home path filled in.

04
Launch

Make sure Steam is open and logged in, then click Play.
The Feral Interactive options window should appear. Click PLAY.
The first launch may take 30-60 seconds.

Troubleshooting

Game doesn't launch at all

Check the Steam log for errors:

tail -50 ~/.steam/debian-installation/logs/console_log.txt

Or use the script: ./shogun2-native-fix.sh --diagnose

"wrong ELF class: ELFCLASS32" in output

This is normal and harmless. It appears because some 64-bit wrappers try to load the 32-bit .so files before the 32-bit game process does. The 32-bit linker will load them correctly.

Black screen / software rendering

You're missing the NVIDIA 32-bit GL library. Check:

glxinfo32 | grep "OpenGL renderer"
# If it says "llvmpipe", you have software rendering
sudo apt install libnvidia-gl-$(nvidia-smi --query-gpu=driver_version \
    --format=csv,noheader | cut -d. -f1):i386

SIGSEGV crash on launch

Make sure libc_mprotect.so is in your LD_PRELOAD. The game requests W+X memory pages which modern kernels reject without this shim.

"No result in response — XML must start with result node"

This is the multiplayer/online error. Single-player still works fine — just dismiss the dialog.

How to uninstall

rm -rf ~/.local/share/shogun2-native-fix/

Then remove the launch option from Steam. Nothing was written to /usr/lib or anywhere system-wide. Your other games and applications are completely unaffected.

Technical notes

This fix was developed by trial and error on a real system (Linux Mint 22.3, Ubuntu 24.04 base, NVIDIA RTX 4070, Steam native). The key discoveries that are not documented elsewhere:

  • OpenSSL 1.0.2 must be compiled with no-asm on modern GCC — the assembler version has a duplicate symbol (bn_sub_part_words) with bn_mul.c that causes a link error
  • The build system uses CFLAG not CFLAGS — passing CFLAGS= on the make command line overwrites the internal flags including the -I paths, causing e_os.h: No such file or directory
  • The Feral binary already includes private_symbol_hack.so as a fix for __libc_dlopen_mode — it just needs to be copied to a path without spaces
  • The GAME_LAUNCH_PREFIX environment variable is read by Shogun2.sh and prepended to the actual binary invocation — this is the correct way to inject LD_LIBRARY_PATH for the 32-bit game without affecting the 64-bit wrapper scripts
  • Steam must be running when you launch — the Feral launcher pings Steam via steam://run/34330 before starting the game
  • libgconf-2.so.4 was removed from Ubuntu 22.04. A stub with ~20 empty function definitions is sufficient

This guide is based on a working installation confirmed in May 2026. The native Linux port of Shogun 2 was abandoned by Feral Interactive years ago and is officially unsupported. If you find improvements or corrections, please share them.
